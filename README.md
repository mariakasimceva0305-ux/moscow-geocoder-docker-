# Moscow Geocoder

Сервис геокодирования московских адресов. Решение нормализует входной адрес, сопоставляет его со справочником зданий и возвращает координаты и ранжированный список совпадений.

## Назначение

По свободному текстовому адресу находить соответствующий объект в справочнике, даже если адрес содержит сокращения, опечатки или неполную запись улицы и номера дома.

## Архитектура

```text
Входной адрес
      │
      ▼
Нормализация текста
      │
      ▼
Точное сопоставление
      │
      ├─ если совпадение найдено
      │
      ▼
Нечёткое сопоставление
      │
      ▼
Ранжирование кандидатов
      │
      ▼
Координаты и список совпадений
      │
      └─ API FastAPI
```

## Что реализовано

- нормализация входных адресов;
- базовый и расширенный контуры сопоставления;
- точное и нечёткое сопоставление;
- ранжирование найденных кандидатов;
- API для запросов геокодирования;
- сценарий оценки качества.

## Структура репозитория

- `src/data_loader.py` — загрузка справочника зданий;
- `src/normalize.py` — нормализация адреса;
- `src/geocode_basic.py` — базовый контур сопоставления;
- `src/geocode_improved.py` — расширенный контур сопоставления;
- `src/api.py` — API;
- `scripts/run_basic_cli.py` — запуск базового контура;
- `scripts/run_improved_cli.py` — запуск расширенного контура;
- `scripts/run_evaluate.py` — оценка качества;
- `ARCHITECTURE.md`, `DOCKER.md`, `SCORE_EXPLANATION.md` — технические материалы;
- `docker-compose.yml` — запуск сервиса в контейнерах.

## Оценка качества

Основные метрики:

- доля корректно найденных адресов;
- качество текстового совпадения;
- географическая ошибка;
- доля попаданий в верхние позиции списка кандидатов;
- задержка ответа API.

## Результаты

CLI и API (`docker compose up`) работают из коробки. Сценарий `scripts/run_evaluate.py` считает метрики при наличии файла `moscow_buildings.csv` в корне (крупный справочник в репозиторий не входит из‑за объёма; порядок подключения данных — в `GITHUB_SETUP.md`). Итоги прогона смотрите в сгенерированных отчётах и в пояснениях к метрикам в `SCORE_EXPLANATION.md`.

## Запуск

```bash
pip install -r requirements.txt

python scripts/run_basic_cli.py "<адрес>"
python scripts/run_improved_cli.py "<адрес>"
python scripts/run_evaluate.py
```

Контейнерный запуск:

```bash
docker compose up --build
```

## Ограничения

- качество зависит от полноты и единообразия адресного справочника;
- редкие сокращения и нестандартные формулировки требуют расширения правил нормализации;
- для промышленного применения фиксируют единый отчёт с итоговыми метриками после каждого прогона оценки.
