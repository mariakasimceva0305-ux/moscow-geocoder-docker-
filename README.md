# Сервис сопоставления московских адресов

## Кратко
Сервис сопоставления адресных строк с адресным справочником. В репозитории собраны базовый и улучшенный конвейеры, API, контейнерный запуск и материалы по архитектуре и оценке качества.

## Задача
Пользовательский ввод адреса часто содержит сокращения, ошибки, пропуски и неоднозначные формулировки. Задача сервиса — привести такую строку к максимально точному совпадению с адресным справочником и сделать это воспроизводимо и удобно для интеграции.

## Что улучшено
- добавлена более аккуратная нормализация адресных строк;
- сопоставление разбито на несколько последовательных этапов вместо одного грубого сравнения;
- повышена устойчивость к сокращениям, опечаткам и неполным адресам;
- сервисная упаковка делает запуск и интеграцию воспроизводимыми.

## Архитектура
```text
[входная адресная строка]
            |
            v
[нормализация и очистка]
            |
            v
[базовое сопоставление]
            |
            v
[дополнительные правила и уточнение]
            |
            v
[лучшее совпадение в справочнике]
            |
            v
[API и контейнерный запуск]
```

## Структура репозитория
```text
src/                основная логика сервиса
scripts/            вспомогательные сценарии
static/             статические материалы
ARCHITECTURE.md     отдельное описание архитектуры
DOCKER.md           детали контейнерного запуска
SCORE_EXPLANATION.md объяснение критериев оценки
```

## Критерии качества
- текстовое сходство;
- географическая ошибка;
- число успешных сопоставлений;
- задержка ответа.

## Ключевой вывод
Улучшенный конвейер даёт более устойчивое сопоставление по сравнению с базовым режимом, особенно на строках с ошибками, сокращениями и неполными данными. Главный практический эффект — рост доли корректных совпадений при сохранении воспроизводимого запуска через контейнеры.

## Запуск
```bash
docker build -t moscow-geocoder .
docker run --rm -p 8000:8000 moscow-geocoder
```

## Ограничения
- результат зависит от полноты и качества адресного справочника;
- на сильно испорченных строках требуется дополнительная логика восстановления структуры адреса;
- для промышленного контура нужен отдельный мониторинг качества и логирование ошибок сопоставления.
